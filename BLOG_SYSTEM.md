# Blog Automation System — Spec & Decisions

## Задача
Щодня автоматично публікувати унікальну статтю на блог агенції.
Тема: **саммарі IT, AI та Tech новин за вчорашній день**.

---

## Строгі правила (відхилення НЕПРИПУСТИМЕ)

1. Стаття виходить **щодня без винятків**
2. Тільки новини **строго за вчорашній день** (CEST timezone)
3. Новини старші ніж вчора — **заборонені**
4. Заголовок — **завжди унікальний**, залежить від конкретних новин дня
5. **Лог тем** — обов'язковий, перевірка конфліктів перед кожною публікацією
6. Стиль статті — **визначається автоматично**, ніколи не фіксований
7. Мова — **тільки англійська**
8. Якщо мало новин — публікувати з тим що є
9. **Конфлікт** = та сама подія повторно → заборонено
10. **НЕ конфлікт** = новий розвиток тієї ж події → дозволено (continuation)

---

## Технічний стек

- **GitHub Action** — запуск щодня о 7:00 CEST (5:00 UTC)
- **Python** — скрипт генерації
- **Claude Haiku 4.5** (`claude-haiku-4-5-20251001`) — conflict detection, вибір стилю, генерація статті
- **Astro** — статичний сайт
- **Vercel** — авто-деплой при push в main

**Вартість:** ~$0.80/1M вхідних токенів. При 3 запитах на день — менше $1 на місяць.

## Система визначення стилю (вирішено)

Claude Haiku аналізує зібрані новини і **самостійно** обирає стиль:

| Стиль | Коли застосовується |
|-------|---------------------|
| **News Roundup** | Багато рівнозначних новин |
| **Deep Dive** | Одна велика подія |
| **Rumor Report** | Витоки, неперевірена інформація |
| **Quick Hits** | Мало новин, короткий формат |
| **Trend Analysis** | Кілька пов'язаних новин утворюють тренд |
| **Breaking Down** | Складне технічне оголошення яке треба пояснити |

Критично: один стиль не може повторюватись кілька днів поспіль.

## Система виявлення конфліктів тем (вирішено)

**Зберігання:** `topic-log.json` у репо, оновлюється після кожної публікації.

**Структура запису:**
```json
{
  "date": "2026-05-09",
  "title": "...",
  "slug": "...",
  "style": "Deep Dive",
  "forced": false,
  "topic_summary": "..."
}
```

**Перевірка:** перед генерацією Claude Haiku отримує останні 30 записів і виносить вердикт:

- **CONFLICT** → та сама подія, не публікувати цю тему
- **CONTINUATION** → новий розвиток старої теми → дозволено, можна згадати попередню статтю
- **NEW** → нова тема → публікувати

**Логіка тонкої грані:**
- iOS 26.4 вчора + iOS 26.4 сьогодні = **CONFLICT**
- iOS 26.4 вчора + iOS 26.4.1 сьогодні = **CONTINUATION**
- iOS 26.4 вчора + знайдена вразливість в iOS 26.4 = **CONTINUATION**

## Повний флоу

```
GitHub Action (7:00 CEST)
  → Fetch 6 RSS джерел
  → Фільтр: pubDate = вчора (00:00–23:59 CEST)
  → Dedupe по URL та заголовку
  → Claude Haiku: перевірка конфліктів (topic-log.json) → вибір стилю → генерація статті
  → Оновлення topic-log.json
  → Commit + push → Vercel деплоїть
```

## Frontmatter schema (вирішено)

Обов'язкові (Astro падає без них): `title`, `description`, `publishedAt`, `category`, `readingTime`

Генеруємо завжди: `tags`, `metaTitle`, `metaDescription`, `keyTakeaways`, `faq`

Фіксовані в скрипті (не генеруються):
```
author: "Bohdan Shvchk"
authorRole: "Founder & Shopify Developer"
authorBio: "..."
authorLinkedIn: "https://www.linkedin.com/in/bogdan-shevchuk-b3827414a/"
```

---

## Continuation-статті (вирішено)

- `topic-log.json` зберігає `slug` кожної опублікованої статті
- При CONTINUATION: Claude Haiku отримує slug + title попередньої статті
- Посилання вставляється органічно в тіло статті ("As we covered yesterday, ...")
- Зміни в схему не потрібні

---

## Недоступність Claude Haiku (вирішено)

- Спроби 1-3: паузи 5с → 15с → 30с
- Спроби 4-6: паузи по 1 годині кожна
- Починаючи з 3-ї спроби і кожна наступна: Telegram сповіщення
- Якщо всі 6 провалились: день пропускається, фінальне Telegram сповіщення

---

## Всі теми дня — конфлікти (вирішено)

Ієрархія рішень:
1. Є CONTINUATION → використати його
2. Тільки CONFLICT → взяти тему з найменшою схожістю, опублікувати
3. Випадок логується в `topic-log.json` з полем `"forced": true`
4. Telegram сповіщення з переліком конфліктних тем

---

## Сповіщення (вирішено)

Канал: **Telegram bot**

Потрібні secrets в GitHub Actions:
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

Сповіщення надсилаються при:
- Кожна спроба підключення до Claude Haiku починаючи з 3-ї
- День пропущено (всі 6 спроб провалились)
- Немає новин за вчора
- Невалідний frontmatter (яке поле відсутнє)
- Всі теми дня — конфлікти (з переліком тем)
- Успішна публікація — з заголовком статті

Формат: строгий технічний текст, без емодзі.
