## 🚨 Найвищий пріоритет

### [AUTO-1] Remote Trigger зупиняється і питає користувача замість автономного виконання
- **Де:** Claude Code → Routines → Daily Morning Trigger (промпт рутини)
- **Проблема:** Агент натрапляє на перешкоду (немає дозволу на push, токен, etc.) і замість вирішення — зупиняється і пропонує "Варіант А / Б / В". О 7:00 нікого немає щоб відповісти. Система не є автоматичною.
- **Рішення:**
  1. Переписати промпт рутини — дати чіткий алгоритм "якщо X → роби Y", без місця для питань
  2. Додати в промпт: "Ніколи не питай користувача. Вирішуй самостійно або завершуй з помилкою."
  3. Дати потрібні дозволи в `settings.json` (git push, git commit, тощо)
- **Потрібно:** Показати поточний промпт рутини щоб переписати його правильно

- **Проблема:** Токен `ghp_*` збережений відкритим текстом. Навіть якщо файл тепер в `.gitignore`, він міг бути в ранніх комітах.

### [BUG-1] Slug обрізається посередині слова
- **Файл:** `scripts/generate_blog.py:81`
- **Проблема:** `text[:60]` ріже без урахування меж слів → URL типу `.../ai-powered-tools-are-transfor`
- **Фікс:**
  ```python
  if len(slug) > 60:
      slug = slug[:60]
      last_dash = slug.rfind('-')
      if last_dash > 20:
          slug = slug[:last_dash]
  ```
- **Статус:** `[ ] TODO`

---

### [BUG-2] Нема валідації frontmatter перед записом файлу
- **Файл:** `scripts/generate_blog.py:139-152`
- **Проблема:** Якщо Groq генерує `title: 'single quotes'`, неправильний формат дати або відсутнє поле — файл записується, Action пушить, а Astro падає при білді на Vercel.
- **Фікс:** Додати валідацію після генерації:
  - Перевірити що є відкриваючий і закриваючий `---`
  - Перевірити обов'язкові поля: `title`, `description`, `publishedAt`, `category`, `readingTime`
  - Перевірити формат `publishedAt: YYYY-MM-DD` (без лапок)
  - Якщо не ок — не писати файл, `sys.exit(1)`
- **Статус:** `[ ] TODO`
